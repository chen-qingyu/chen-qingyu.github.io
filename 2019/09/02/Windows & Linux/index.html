Windows & Linux

发表于 2019-09-02 更新于 2019-09-03 分类于 Others ：

有位禅宗大师提出参禅的三重境界：参禅之初，看山是山，看水是水；禅有悟时，看山不是山，看水不是水；禅中彻悟，看山仍是山，看水仍是水。殊不知，对于工具的使用也是如此的。

看山是山，看水是水

以前刚听说Linux的时候，给人一种神秘的印象，觉得Linux似乎只有黑客那种级别的人才配得上，普通人还驾驭不了。于是出于好奇心，我开始折腾Linux。

最开始给虚拟机装的是CentOS，然后上网查资料，看教程，学命令，搞配置，编脚本……

然后又给虚拟机装了Ubuntu，继续上网查资料，看教程，学命令，搞配置，编脚本……

热衷于yum update和apt-get update的我，在用了几次vim+gcc写C语言程序之后，发现还没有Windows下的Sublime Text好用，不得不说，vim的键位设置实在是有点反人类，或许是我对击键速度尚且没有很高的要求，有“编辑器之神”称号的vim并没有让我觉得很好用。我又准备试一下有“神之编辑器”称号的Emacs，但是其臃肿的体型和反人类的键位再次劝退了我……这时候，我开始回归Windows了。

看山不是山，看水不是水

回归Windows后，发现Windows下也有类似Linux shell的环境，就是Cygwin，还有Git for Windows也很好用，解决了每次用Linux shell都得开虚拟机的麻烦，而且我个人觉得Sublime Text比vim和Emacs都好用，虽然后两个可定制化程度超级高，但其学习曲线也是很迷的……

微软大法好！

看山仍是山，看水仍是水

折腾到最后，明白了一个道理：工具是用来提高效率的，而非用来炫耀的。

其实，不管是Windows也好，还是Linux也好，本质不过是一种工具而已。

有些人用Linux是觉得真的好用，因为他们已经把Linux配置得很适合自己了；有些人用Linux只是为了炫耀，比如之前的我。当用客观的眼光去看待这两种操作系统时，其实并不能说谁比谁好，也并非普通人用Windows而程序员用Linux。随着开源和跨平台的流行，这两者之间的界限已经不那么泾渭分明了。

要我说，微软其实像一位严格的家长，把各种细节都给你安排得妥妥的，但你因此失去了部分自由；而Linux像一位活泼的伙伴，喜欢把你拉着和她一起玩，但有时候玩坏了你要负责要能通过自己的技术把问题解决了。

最后，推荐一下Deepin，因为其一它是国产的，其二它性能软件支持各方面确实做得不错。
