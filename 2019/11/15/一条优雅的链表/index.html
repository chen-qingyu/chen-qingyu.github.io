一条优雅的链表

发表于 2019-11-15 更新于 2019-11-16 分类于 Data Structure 5.8k

线性表(Linear List)的ADT：

对象集：$n(n \ge 0)$ 个元素构成的有序序列 $a_1, a_2, a_3 \cdots a_n$
操作集：线性表 list 属于 list_t ，整数 i 表示元素下标(从0开始)，元素 x 属于 element_t ，基本操作有：

  • list_t CreateList(void)
  • list_t FindByIndex(int i, list_t list)
  • list_t Find(element_t x, list_t list)
  • bool Insert(element_t x, int i, list_t list)
  • bool Delete(int i, list_t list)
  • int GetLength(list_t list)
  • void Print(list_t list)
  • bool LinkList(list_t list1, list_t list2)

程序说明：

  1. 这是一个我写来练习的程序，因此并没有实现所有的链表操作，并且在函数内部有充分的提示信息。
  2. 更改链表的函数有bool类型的返回值，因此可以通过if语句方便判断更改是否成功。
  3. 有头结点，头结点的数据域储存链表长度。因此可以将GetLength()的时间复杂度从 $O(N)$ 降到 $O(1)$ 。
  4. 变量命名很清晰。C语言不具有面向对象的特性，因此这里函数采用大驼峰命名法，变量采用小驼峰命名法。别小看这一点，好的变量命名一看就懂，比如GetLength()表示获取长度，LinkList()表示连接链表，new表示待插入的新结点，del表示待删除的结点，tail表示尾结点等等。网上太多代码的变量命名都乱七八糟的。
  5. Print()从原来的依据指针遍历改为了依据长度遍历。这样的话即使是循环链表，也可以很方便地打印。
  6. LinkList()依据传入的参数连接两个链表或是制作循环链表。事实上，把一个链表头尾连接起来就是循环链表了，我在只学过单链表的情况下自己想出来并且写出来了，而且还实现得很优雅，我太棒了哈哈哈。
  7. 大家都知道，malloc()和free()一定要成对出现、一一对应。但是我看网上好多人都不知道free()和p = NULL也一定要成对存在，也就是说释放一个（全局）指针变量p指向的内存空间后要立刻把它指向NULL。为什么呢？因为释放前后，p所指向的内存空间是一样的。所以释放后p所指向的仍然是那块内存空间。可是释放后的那块内存空间已经不属于p了，该内存空间可能会被分配给其他函数使用。如果其他函数在那块内存空间存放了值，而你现在不小心再用p往里面写入了数据就会把那个值给覆盖了。所以当内存空间被释放后，要立刻把指针变量指向NULL。
  8. 使用malloc()申请了内存空间之后一定要紧接着用if (p == NULL)判断申请成功没。好多人申请就申请了，都不检查一下，虽然一般不会出错，可是保持良好的编程习惯以后写程序不容易出Bug。
